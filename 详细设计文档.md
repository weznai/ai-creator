### 1.2 技术选型

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | React 18 + TypeScript + Ant Design | 主流技术栈，生态完善 |
| 后端 | Node.js (NestJS) / Java (Spring Boot) | 高性能、易维护 |
| 数据库 | MySQL 8.0 | 主数据存储 |
| 缓存 | Redis 7.0 | 缓存、会话、消息队列 |
| 对象存储 | MinIO / 阿里云OSS | 图片、视频存储 |
| 向量数据库 | Milvus | RAG语义检索 |
| 网关 | Nginx / Kong | 反向代理、负载均衡 |
| 容器化 | Docker + Kubernetes | 部署、编排 |

## 2. 数据库详细设计

### 2.1 用户模块

#### 2.1.1 用户表 (users)

```sql
CREATE TABLE `users` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '用户ID',
  `phone` VARCHAR(11) NOT NULL COMMENT '手机号',
  `nickname` VARCHAR(50) DEFAULT NULL COMMENT '昵称',
  `password` VARCHAR(255) DEFAULT NULL COMMENT '密码(bcrypt加密)',
  `avatar` VARCHAR(500) DEFAULT NULL COMMENT '头像URL',
  `status` TINYINT NOT NULL DEFAULT 1 COMMENT '状态:0-禁用,1-正常',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted_at` DATETIME DEFAULT NULL COMMENT '删除时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_phone` (`phone`),
  KEY `idx_status` (`status`),
  KEY `idx_created_at` (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

## 5. 消息队列设计

### 5.1 队列定义

| 队列名称 | 用途 | 消费者数量 | 重试策略 |
|----------|------|------------|----------|
| image:generate | 图片生成任务 | 5 | 3次,指数退避 |
| video:generate | 视频生成任务 | 3 | 3次,指数退避 |
| text:generate | 文案生成任务 | 5 | 3次,指数退避 |
| publish:immediate | 立即发布任务 | 10 | 3次,指数退避 |
| publish:scheduled | 定时发布任务 | 5 | 3次,指数退避 |

### 5.2 消息格式

```json
{
  "taskId": 12345,
  "taskNo": "IMG202401150001",
  "taskType": "text2img",
  "userId": 1,
  "modelId": 1,
  "modelCode": "dall-e-3",
  "provider": "openai",
  "prompt": "一只可爱的猫咪",
  "negativePrompt": "模糊",
  "width": 1024,
  "height": 1024,
  "generateCount": 4,
  "callbackUrl": "https://api.example.com/callback/image",
  "createdAt": "2024-01-15T10:30:00Z",
  "retryCount": 0
}
```

### 5.3 重试策略

```
重试间隔计算: delay = baseDelay * 2^retryCount

第1次重试: 60秒后
第2次重试: 120秒后
第3次重试: 240秒后

超过3次重试后,任务标记为失败,需要人工处理
```

interface ContentLimits {

  supportedVideoFormats: string[];

### 8.2 小红书适配器实现

```typescript
class XiaohongshuPublisher implements PlatformPublisher {
  readonly platform = 'xiaohongshu';
  readonly platformName = '小红书';
  
  private readonly apiConfig = {
    baseUrl: 'https://open.xiaohongshu.com',
    appId: process.env.XHS_APP_ID,
    appSecret: process.env.XHS_APP_SECRET
  };
  
  getContentLimits(): ContentLimits {
    return {
      maxTitleLength: 20,
      maxContentLength: 1000,
      maxMediaCount: 18,
      maxVideoDuration: 900,
      supportedImageFormats: ['jpg', 'jpeg', 'png', 'webp'],
      supportedVideoFormats: ['mp4', 'mov']
    };
  }
  
  adaptContent(content: PublishContent): PublishContent {
    const limits = this.getContentLimits();
    
    return {
      ...content,
      title: content.title?.substring(0, limits.maxTitleLength),
      content: content.content.substring(0, limits.maxContentLength),
      mediaUrls: content.mediaUrls.slice(0, limits.maxMediaCount)
    };
  }
  
  async publish(account: PlatformAccount, content: PublishContent): Promise<PublishResult> {
    const adaptedContent = this.adaptContent(content);
    
    // 1. 上传媒体文件
    const mediaIds = await this.uploadMedia(account, adaptedContent.mediaUrls);
    
    // 2. 发布内容
    const response = await this.request(account, '/api/v1/note/create', {
      method: 'POST',
      body: {
        title: adaptedContent.title,
        content: adaptedContent.content,
        media_ids: mediaIds,
        tags: adaptedContent.tags,
        post_time: adaptedContent.scheduledAt
      }
    });
    
    return {
      success: true,
      postId: response.note_id,
      postUrl: `https://www.xiaohongshu.com/explore/${response.note_id}`
    };
  }
}
```
